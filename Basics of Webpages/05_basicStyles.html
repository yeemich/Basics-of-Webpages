5.2. Units

There are a number of different units so I'll quickly go over all of them.

Let's start with the fixed units which don't change in length.

  • cm (centimeters)
  • mm (millimeters)
  • in (inches)
  • px (pixels)
  • pt (point)
  • pc (picas)

Of those units, the main one to remember would be px, the other ones are not often used.

The next set of units are relative units which do change in length.

  • em (current font-size)
  • ex (height of a lowercase 'x' character)
  • ch (width of the '0' character of current font-size)
  • rem (font-size of the "html" tag)
  • vw (1% of the width of the webpage/veiwport)
  • vh (1% of the height of the webpage/veiwport)
  • vmin (1% of the webpage's/veiwport's smaller dimension. ex: phone:width, tv:height)
  • vmax (1% of the webpage's/veiwport's bigger dimension. ex: phone:height, tv:width)
  • % (relative to the dimensions of the parent/container element

Of these units, the main ones to remember are em, vw, vh and %.

Those are all the units of CSS. If you can just remember the main 5 that I mentioned, you should be good to do 99% of the dimensioning you need to do for your website. If you want to go into a bit more detail about the units, you can check out w3school's page on CSS units or the pdf on CSS units by w3. Both are very helpful and good to look at for any extra clarity needed on CSS units.

6. Font-Family: Let's you pick the font you want. In case a device doesn't have the font you want, it's best to put your font-family as a list of possible fonts that you would be ok with the computer displaying in case they don't have the your first choice available. At the end of the list, you should put a few common fonts in which you know will work with most devices that you would like.
