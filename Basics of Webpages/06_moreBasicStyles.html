More Basic Styles

In this file, we'll go over the rest of the basic styles that you should know in CSS. Similar to the last file, you should make sure to read what's in the style tag above to see why everything on the page has the style it has.

  1. Height
  2. Width
  3. Margin
  4. Padding
  5. Border
  6. Text-Align
  7. Float

Now let's step away from fonts for a bit.

1. height: sets the height of any non-inline element. This one is set to 90px

2. width: sets the width of any non-inline element. This one is set to 300px

You can define height and width using any of the units we just went over.

Also, it may be weird that height and width can't be set in an inlined element. It actually makes a lot of sense though. The whole point of inline is to take up the least amount of space possible, so if you're just going to set the width and height, what's the point?

I've put all tags for the next two styles in a div so you can see them better

3. Margin: Increases the size of the tag from the outside. Here's 60px. I'm going to set the margin of the hr tags to 0 so you can see it better and don't get the default margin of the hr tags mixed up with the margin of the p tags.

You can also control the vertical margin separate from the horizontal margin. Here's 20px vertical and 200px horizontal.

You can also control the margin above, below, left and right individually. Here's 20px above, 40px to the right, 60px below and 150px to the left.

If you just wanted to change the margin in one direction, you can just write "margin-" and then the direction you want. Here's 50px to margin-left. As you can see, the paragraph tag has some margin by default so by only changing the margin in the left direction, the margin for the other three directions stay as the default value for the paragraph tag.


4. Padding: Increases the size of the tag from the inside. Here's 60px.

You can also control the vertical padding separate from the horizontal padding. Here's 20px vertical and 200px horizontal.

You can also control the padding above, below, left and right individually. Here's 20px above, 40px to the right, 60px below and 150px to the left.

If you just wanted to change the margin in one direction, you can just write "padding-" and then the direction you want. Here's 50px to padding-left.

Also remember that margin and padding take in lengths with units, so any of the units we went over will also work with margin and padding.

5. Border: This simply sets attributes of the border around a tag. By default on most tags, there's no border so we'll bring one up to show you what they look like.


style:

First we need to select the thickness of the border. This will take a number with units so like I said before, all units will work but we'll stick with pixels.

style: 5px

Now we need to set the style of the border. A border can (is "can" an extra word here?) style is very similar to a text-decoration style. Here is a list of possible styles:

  1. Dotted
  2. Dashed
  3. Solid
  4. Double
  5. groove
  6. ridge
  7. inset
  8. outset
  9. none (meaning that there is no border)
  10. hidden (meaning that the border takes up space, but is invisible)

I rarely use anything other than the solid border so I'm going to stick with that one for now, but you can check out this site here to see a short description of each border along with what they look like.

style: 5px solid

Now the last thing we need to do is set the color of the border and we're done.

style: 5px solid darkblue

I'll show you a few quick combinations of borders you can make with this.

  1. 4px inset lightcoral
  2. 10px dotted cyan
  3. 6px ridge grey
  4. 15px double green

You can also set (the?) side of the border to a different border using (maybe try: "You can also set (the?) side of the border to a different style using....", though I am only suggesting this to use the word "border" less, but it's okay how it is) "border-" and the direction you want. Very similar to padding and margin.

border-top: 5px dotted blue

You can also set each style individually. Very similar to text decoration and Font.

border-width: 10px, border-style: groove and border-color: chocolate;

You can also define a single attribute in three different ways, which I'll go over with an example where I define width, style and color all using the different methods.

The first and second way are definitely the best way for defining each side of a border individually, but if you can't remember or don't understand how it works, the last method should be clear and easy to remember.

Now we're going to talk about two more styles and we're done.

6. Text Align: lets you align the text within a non-inline element. The possible ways you can align a the (omit "a" or "the" here) text in a non-inline element are the following:

  • text-align: left (default)
  • text-align: right
  • text-align: center
  • text-align: justify (though it may look similar to "text-align: left" at first, you may notice that if we add enough text to it and set it to "text-align: justify", it'll actually make sure that each line ends exactly at the end of the screen, unless it can't make it to the edge of the screen (not exactly sure what you mean here). This is similar to the effect of a newspaper heading.

Now you may have noticed that I said it only works for non-inline elements. This is for a similar reason (missing some words here?) the height and width. (Also, if you're curious about how the styles are working, CSS reads linearly so you can overwrite (or override?) values after setting them in the code if you'd like to do so)

In a block element, it takes up as much of the width of the screen it can get (possible?) so that the text inside has a lot of freedom to move.

In an inline element, it tries to be as small as possible so the text can't move around at all. causing text-align to be useless.

7. Float: moves the floated element to the top of its container, then it moves to one side and finally forces elements to go on the other side of it. Let's do a few examples.

It may have looked normal at first, but as you can see from this paragraph tag, which should've taken up the enitre width of the screen by default, it now aims to swerve all text inside of it around the floated element. This is what float does.

Here is what happens when I set a div to float right.

You may also notice that if I do everything the exact same as the last one, but instead of a span tag (inline-element), we use a div (block-element), it does the exact same thing. This is because when we set the float value of an element, it starts to act differently from what its display is set to.

The big difference between a regular block element and block element which has been floated, is if the block element doesn't have a specified width, it will act like an inline element.

In other words, by default, a floated block-element will take up the least amount of space possible, which is completely against all that a block element is. (You could try: "..., which is completely against the definiton of a block element")

Though, if you do want to have a specified width, you must use a div element because inline elements still won't let you change the height and width. That's the main difference between a floated block-element and a float inline-element.

Float has three main options

  1. left
  2. right
  3. none (default

There's also a style called "clear" which negates the effects of float, but we won't talk about that here.
